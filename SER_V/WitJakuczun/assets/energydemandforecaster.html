1 Wstęp

Niniejszy dokument opisuje przykładowe użycie modułu generowania prognoz zapotrzebowania na energię (KSE).

2 Przygotowanie środowiska pracy

2.1 Pakiety

2.1.1 Pakiety wymagane przez pakiet energydemandforecaster

Będziemy potrzebowali następujących pakietów:

  • data.table
  • lubridate
  • EnergyDataRepository
  • redrepoadapter
  • weatherforecaster
  • energydemandforecaster

Pakiety ładujemy poniższymi instrukcjami

library(data.table)
library(lubridate)
library(EnergyDataRepository)
library(edrepoadapter)
library(weatherforecaster)
library(energydemandforecaster)

2.1.2 Dodatkowe pakiety

Na potrzeby niniejszej prezentacji możliwości będziemy potrzebowali następujących pakietów.

  • ggplot2
  • plyr
  • reshape2

Pakiety ładujemy następującymi instrukcjami:

library(ggplot2)
library(plyr)
library(reshape2)

2.2 Repozytorium danych energetycznych

Aby przykłady z niniejszej prezentacji zadziałały potrzebujemy aby w naszym repozytorium były odpowiednie dane:

  • Zapotrzebowanie energetyczne kraju (KSE)
  • Dane pogodowe

Poniższy kod pokazuje w jaki sposób powinniśmy zaktualizować nasze repozytorium.

edrepo <- edrepo.buildEnv(edrepo.path)
edrepo.uploadData(
    names = c("ZAP_KSE"),
    from = "2002-01-01",
    till = "2013-12-31",
    env = edrepo)

edrepo.uploadData(
    name = "Pogodowe",
    from = "2002-01-01",
    till = "2013-12-31",
    env = edrepo)

Uwaga

Aktualizacja repozytorium jest wymagana tylko i wyłącznie jeśli nie posiadamy w repozytorium odpowiednich danych.

Uwaga

Na potrzeby całej prezentacji zakładamy, że repozytorium znajduje się podkatalogu edrepo w aktualnym katalogu roboczym. W ogólnym przypadku repozytorium może być w dowolnym miejscu.

3 Budowanie modelu

Poniżej opisujemy, krok po kroku, jak zbudować model prognostyczny dla zapotrzebowania na energię elekryczną. Budowa modelu składa się z następujących kroków:

  1. Przygotowanie danych
  2. Zbudowanie modelu

3.1 Przygotowanie danych

Aby zbudować model należy dostarczyć dwie tabele danych:

  1. holidays - jest to ramka danych o jednej kolumnie o nazwie data zawierająca daty (w formacie POSIXct) świąt.
  2. train.data - jest to ramka danych o zawierająca następujące informacje:
    • Temperatura [stopnie Celsjusza]
    • Historia zapotrzebowania energetycznego kraju

3.2 Budowa modelu

Aby zbudować model należy skorzystać z funkcji demand.model z pakiety energydemandforecaster. Zacznijmy od przeczytania dokumentacji dla tej funkcji

help(demand.model)

Zbudujmy model poniższą instrukcją:

model <- demand.model(
    train.data = demand.input.data$train.data[ date < as.Date("2013-01-01"),],
    holidays = demand.input.data$holidays)

Uwaga

Model zbudowaliśmy dla okrojonego zestawu danych treningowych poprzez wyrzucenie całego roku 2013. Zrobiliśmy to aby pokazać jak efektywny jest nasz model prognostyczny.

3.3 Diagnostyka

Model jest obiektem typu list z dodanym atrybutem class informującym, że jest to model zapotrzebowania energetycznego kraju.

names(model)
[1] "daily.model"    "daily.profiles"
class(model)
[1] "demand.model"

Jak widać model składa się z dwóch komponentów:

  • daily.model - model prognostyczny dla średnio-dobowego zapotrzebowanai energetycznego kraju.
  • daily.profiles - profile dla poszczególnych typów dni tygodni.

3.3.1 Profile dla typów dni tygodnia

Zobaczmy jak wyglądają profile dla poszczególnych dni tygodni dla różnych miesięcy:

print(
    ggplot(model$daily.profiles,
           aes(x = hour, y = demand, color = factor(month))) +
    geom_point() +
    geom_line() +
    facet_wrap(~ day.type))

Zobaczmy jak wyglądają profile dla poszczególnych dni tygodni w ramach każdego miesiąca:

print(
    ggplot(model$daily.profiles,
           aes(x = hour, y = demand, color = factor(day.type))) +
    geom_point() +
    geom_line() +
    facet_wrap(~ month))

4 Generowanie prognoz

W tej części pokażemy jak mając model prognostyczny zapotrzebowania energetycznego kraju można generować prognozy. Generowanie prognoz składa się z następujacych kroków:

  1. Ustalenie jaki horyzont prognozy nas interesuje.
  2. Przygotowanie prognoz predyktorów: temperatury oraz produktu krajowego brutto.
  3. Wygenerowanie prognozy.

4.1 Horyzont prognozy

Zdefiniujmy horyzont prognozy na 366 dni.

n.ahead <- 366

4.2 Przygotowanie prognoz predyktorów

4.2.1 Generowanie prognozy temperatury

Prognozę temperatury wykonamy przy wykorzystaniu pakietu weatherforecaster. Pakiet ten dostarcza funkcje temperature.model, która tworzy model prognostyczny dla temperatury. Korzystając z tej funkcji wygenerujemy prognozę

temp.pred <- predict(
    temperature.model(history = demand.input.data$train.data[date < as.Date("2013-01-01"), list(date, temperature)]),
    start.date = as.Date("2013-01-01"),
    n.ahead = n.ahead)

    n.ahead = n.ahead,
    xregs = data.frame(
        temperature = temp.pred$temperature.pred,
        gdp = gdp.pred),
